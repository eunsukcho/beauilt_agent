## 저작권 및 데이터 사용 가이드

### ⚠️ 중요: 저작권 주의사항

벡터 DB에 데이터를 추가할 때는 **저작권**을 반드시 확인해야 합니다.

### ✅ 사용 가능한 경우

1. **자신이 작성한 콘텐츠**
   - 직접 작성한 문서, 노트, 가이드
   - 완전히 자유롭게 사용 가능

2. **공개 도메인 (Public Domain)**
   - 저작권이 만료된 자료
   - 저작권 포기 선언이 있는 자료

3. **Creative Commons 라이선스**
   - CC BY (출처만 명시하면 사용 가능)
   - CC BY-SA (출처 명시 + 동일 라이선스)
   - CC BY-NC (비영리 목적만 가능)
   - CC0 (완전 자유 사용)

4. **명시적 허가를 받은 경우**
   - 저작권자로부터 직접 허가를 받은 경우

5. **학술 논문 (비영리 목적)**
   - 연구/학습 목적의 인용
   - 출처 명시 필수
   - 상업적 사용은 별도 허가 필요

### ❌ 사용 불가능한 경우

1. **저작권이 있는 블로그/웹사이트 콘텐츠**
   - 허가 없이 사용 시 저작권 침해 가능
   - 출처를 명시해도 허가 없이는 사용 불가

2. **상업적 콘텐츠**
   - 제품 설명서, 브랜드 가이드
   - 마케팅 자료

3. **명확한 라이선스가 없는 콘텐츠**
   - "모든 권리 보유" 표시가 있는 경우

### 💡 안전한 대안

1. **직접 작성**: 자신의 지식과 경험을 바탕으로 작성
2. **공개 도메인 자료 활용**: 위키피디아, 정부 공개 자료
3. **참고만 하고 재작성**: 여러 출처를 참고하여 자신의 언어로 완전히 재작성

### 🏥 학회/학술 자료 사용 가이드

**대한피부과학회 자료의 경우:**

✅ **사용 가능한 경우:**
- 비영리 교육/연구 목적
- 출처 명시 필수
- 인용 형식 준수
- 짧은 인용 (전체 복사 아님)

❌ **주의해야 할 경우:**
- 상업적 사용
- 전체 문서 복사
- 출처 미명시

**권장 방법:**
- 학회 자료를 참고만 함
- 자신의 언어로 완전히 재작성
- 출처는 참고 자료로 명시

### 📰 뉴스 기사 사용 가이드

✅ **상대적으로 안전한 경우:**
- 짧은 인용 (핵심 내용만)
- 출처 명시 필수
- 비영리 교육/연구 목적
- 사실을 자신의 언어로 재서술

❌ **위험한 경우:**
- 전체 기사 복사
- 상업적 사용
- 출처 미명시

```
data/ingredient_cards/
├── 01_히알루론산.md
├── 02_세라마이드.md
├── 03_나이아신아마이드.md
├── 04_레티놀.md
├── 05_비타민C.md
└── ...
```

**카드 구조:**
- 기본 정보 (화학명, 주요 효능, 적합한 피부 타입)

- 주의사항
- 함께 사용하면 좋은 성분
- 피해야 할 조합
- 실전 팁

#### 2. 케어 루틴 가이드 (`data/care_routines/`)

피부 타입별 구체적인 케어 루틴:
- 건성피부_아침루틴.md
- 건성피부_저녁루틴.md
- 지성피부_아침루틴.md
- 등등...

#### 3. 증상별 대응 가이드 (`data/symptom_guides/`)

- 트러블_급성발진.md
- 각질_과다.md
- 등등...

### 📝 데이터 추가 방법

1. 위 폴더 구조대로 새 폴더 생성
2. 마크다운 파일 작성

4. `uv run python scripts/init_vector_db.py` 실행

## 성분 데이터 전략

### 식약처 화장품 원료성분정보 API

**API 정보:**
- **제공기관**: 식품의약품안전처 (MFDS)
- **URL**: https://www.data.go.kr/data/15111774/openapi.do
- **데이터 형식**: JSON/XML
- **비용**: 무료
- **이용허락범위**: 제한 없음 (공공누리)

**제공 데이터 항목:**
- 표준명 (한글)
- 영문명
- CAS No (화학물질 등록번호)
- 기원 및 정의
- 이명 (별칭)
- 분자식 등

### 벡터 DB vs RDB/API 비교

#### 벡터 DB에 적합한 데이터

**✅ 효능/사용법 등 설명적 정보**
- "히알루론산의 효능은?"
- "세라마이드를 어떻게 사용하나요?"
- "나이아신아마이드와 함께 써도 되나요?"

**현재 `ingredient_cards/`에 있는 내용:**
- 기본 정보, 작용 원리, 사용 방법
- 주의사항, 함께 사용하면 좋은 성분
- 실전 팁, 자주 묻는 질문

**이유:**
- 의미적 검색에 적합
- 유사도 검색으로 관련 정보 찾기 가능
- 자연어 질문에 답변 가능

#### RDB/API에 적합한 데이터

**✅ 구조화된 정확한 정보**
- "히알루론산의 CAS No는?"
- "Sodium Hyaluronate의 표준명은?"
- "이 성분의 기원 및 정의는?"

**식약처 API 데이터:**
- 표준명, 영문명, CAS No
- 기원 및 정의
- 이명 (별칭)

**이유:**
- 정확한 매칭이 필요
- 구조화된 데이터 조회
- 실시간 최신 정보 필요

### 권장 아키텍처: 하이브리드 접근법

```
사용자 질문: "히알루론산 효능이 뭐야?"
    ↓
1. 벡터 DB 검색 (ingredient_cards)
   → 효능, 사용법, 주의사항 등 설명적 정보
    ↓
2. 필요시 식약처 API 조회
   → 표준명 확인, CAS No, 기원 및 정의 등 정확한 정보
    ↓
3. 두 정보를 결합하여 답변 생성
```

### 데이터 분리 전략

**벡터 DB (`ingredient_cards/`):**
- ✅ 성분의 효능, 사용법, 주의사항
- ✅ 피부 타입별 적합도
- ✅ 성분 조합 가이드
- ✅ 실전 팁

**RDB/API (식약처 API 연동):**
- ✅ 표준명, 영문명, CAS No
- ✅ 기원 및 정의
- ✅ 이명 (별칭)
- ✅ 최신 정보 (API로 실시간 조회)

### 결론 및 권장사항

**✅ 권장: 하이브리드 접근법**

**벡터 DB에는 넣지 않는 것이 좋습니다.**

**이유:**
1. 데이터 특성 불일치
   - 식약처 API: 구조화된 정확한 데이터
   - 벡터 DB: 의미적 검색에 적합한 텍스트 설명

2. 검색 목적이 다름
   - "히알루론산 효능?" → 벡터 DB (설명적 정보)
   - "히알루론산 CAS No?" → API (정확한 데이터)

3. 현재 구조가 적합
   - `ingredient_cards/`: 효능, 사용법 등 설명적 정보
   - `get_ingredient_data()`: RDB/API에서 정확한 정보 조회

**최종 권장사항:**
- 벡터 DB: 효능, 사용법 등 설명적 정보 (`ingredient_cards/`)
- 식약처 API: 표준명, CAS No 등 정확한 정보 (실시간 조회)
