## 📊 현재 프로젝트 상태

### ✅ 완성된 부분 (코드 작성 불필요)

1. **기본 아키텍처**
   - ✅ LangGraph 워크플로우 구조
   - ✅ 5개 핵심 에이전트 (진단, 분석, 가이드, 시뮬레이션, 성분 분석)
   - ✅ 웹 검색 에이전트 (벡터 DB 보완)

   - ✅ 대화 히스토리 관리

2. **데이터 구조**
   - ✅ 벡터 DB 연동 (Chroma)

   - ✅ 샘플 데이터 (피부관리의기초.md, skin_type_cards, ingredient_cards)

3. **인터페이스**
   - ✅ 커맨드라인 인터페이스 (`main.py`)
   - ✅ Streamlit 웹 인터페이스 (`streamlit_app.py`)

4. **설정 및 환경**
   - ✅ 환경 변수 관리 (`.env`)
   - ✅ uv 프로젝트 설정 (`pyproject.toml`)
   - ✅ 패키지 의존성 관리

### ⚠️ 개발자가 직접 해야 할 작업

#### 1. 필수 작업 (반드시 해야 함)

- [ ] **환경 변수 설정**
  - `.env` 파일에 OpenAI API 키 입력
  - 위치: 프로젝트 루트의 `.env` 파일
  - 예시:
    ```env
    OPENAI_API_KEY=sk-...
    OPENAI_MODEL=gpt-4o-mini
    ```

- [ ] **벡터 DB 초기화**
  - 명령어: `uv run python scripts/init_vector_db.py`
  - 이 작업을 하지 않으면 벡터 검색이 작동하지 않음
  - 현재는 기본 데이터가 포함되어 있음

- [ ] **실행 테스트**
  - 커맨드라인: `uv run python main.py`
  - 웹 앱: `uv run streamlit run streamlit_app.py`
  - 실제로 동작하는지 확인 필요

#### 2. 선택 작업 (필요에 따라)

- [ ] **데이터 추가/수정**
  - `data/피부관리의기초.md` 내용 보강
  - `data/skin_type_cards/` 카드 추가/수정
  - `data/ingredient_cards/` 성분 카드 추가
  - 방법: 파일 수정 후 `scripts/init_vector_db.py` 재실행

- [ ] **말투/스타일 커스터마이징**
  - 각 에이전트의 `system_prompt` 수정
  - 위치: `agents/*_agent.py` 파일의 `__init__()` 메서드
  - 예시: 더 친근하게, 더 전문적으로 등

- [ ] **성능 튜닝**
  - 벡터 검색 개수 조정 (`k` 값)
  - 대화 히스토리 개수 조정
  - 모델 변경 (`.env` 파일에서)

- [ ] **기능 추가**
  - 새로운 에이전트 추가
  - 새로운 데이터 소스 연동
  - UI 개선

#### ✅ 코드 작성 없이 할 수 있는 것들:

1. **바로 사용하기**
   ```bash

   # 2. 벡터 DB 초기화
   uv run python scripts/init_vector_db.py
   
   # 3. 실행
   uv run python main.py
   ```
   → 이 정도면 **바로 사용 가능**합니다!

2. **데이터만 추가하기**
   - `data/` 폴더에 마크다운 파일 추가
   - `scripts/init_vector_db.py` 재실행
   → 코드 수정 없이 데이터만 추가 가능

3. **설정만 변경하기**

   - 각 에이전트의 프롬프트 텍스트만 수정
   → 복잡한 코드 수정 없이 커스터마이징 가능

#### ⚠️ 코드 작성이 필요한 경우:

2. **복잡한 커스터마이징**
